HTML Markup: Tags & Elements

<Name>Contents</Name> ---- ** This is known as an Element ** (Everything within a set of opeing & closing Tags).
<Name> = OPENING TAG
</Name> = CLOSING TAG

Tags are part of basic HTML vocabulary. An HTML Tag is always contained within angled brackets. Most tags have an Open Tag and Closing Tag. Some examples are:

B Tag - <b>contents</b>
b = BOLD (The contents will always appear in bold).

Em Tag - <em>contents</em>
em = EMPHASIS (The content will always appear in italics).

Some tags do not require a closing tag. These tags are called Void Tags. Some expamples are:

<img> = Image Tag
<br> = Break Tag

HTML ATTRIBUTES - A property of an HTML element.

- An HTML Attribute looks like the following:
<Tag Attr ="Value"> Contents </Tag>

- An example of a tag that uses an Attribute is the Anchor Tag.
- Anchor Tag - <a>
- Anchors are for making links.

FULL EXAMPLE: <span ="code"><a href =www.reddit.com>derp</a></span>
** href = Attribute Name
** www.reddit = Value (in this case a URL)
** derp = Contents

IMAGE TAG (<img>)
<img src ="url" alt ="text">
**alt = "text" is required (text gets displayed when image does not load).

Lesson 2: Creating a Structured Document with HTML

HTML & CSS

Front End Web Development - Area of programming that is inherently visable: Any code you write will be visable inside the browser.

HTML

- Has specific language with syntax and rules (you write HTML in a document, except you follow HTML syntax and rules as opposed to grammatical syntax and rules).
- The basic word in HTML language is a Tag. Browsers turn HTML tags into elements that form a "tree" (more on "trees" in the next section).

CSS

- Language used to define the style of the page. Allows you to use syntax and rules of HTML to change how elements appear on the page (Font, Color, Background, etc.).

Summary & Analogy:

As stated above, both HTML and CSS are "languages." HTML controls the structure of a webpage, while CSS controls the style (how it looks). To better understand, here is a helpful analogy:

A website is like a house. HTML is the structure (Where the walls are placed, what room is what, etc...). CSS is the style of your house (What color is the carpeting? What decorations do you have? What color are the walls? etc...). Java Script are your interactive components (your garage door opener, your televeision remote, etc....). They change some element of your house.

The "Tree-Like Structure" of HTML

- Browsers build a "TREE-LIKE STRUCTURE" of the webpage.
- The "Tree-like Structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. In a browser, this structure shows up as a series of nested boxes. These are boxes inside of boxes, inside of boxes, and so on...

* When programmers talk about "DOM" (Document Object Model), they are talking about the "tree-like structure" of a page.
* When looking at "Developer Tools" in a browser, sideways triangles are an HTML element. This can be text or another element, etc.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

- Everything on websites are in square and rectangular boxes.
- All shapes are within a box. They are created by style. Ex: Circles, Triangles, etc. are all within boxes set by code.
- Boxes are defined by DIV tags (a block tag)

Text Editors (for programming)

When writing code, programmers use special Text Editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3: Adding CSS Style to HTML Structure

CSS

There is a way to add style without having to use a separate file and the <link> tag. You can also put your style code in between an opening and closing <style> tag. For Example:

<Style>
.description {
color: red;
}
</Style>

- CSS stands for "Cascading Style Sheets". The "C" is by far the most important part of this phrase.

* To better understand CSS, we must understand Inheritance: A key feature of CSS that relies on the ancestor descendant relationship to operate. Inheritance is the mechanism by which properties are applied not only to a specific element, but also to its descendants. In general, descendant elements inherit text-related properties, but box related properties are not inherited. Properties that can be inherited are color, font, letter-spacing, line height, list style, text align, text indent, text transform, visibility, white-space and word spacing. Properties that cannot be inherited are background, border, display, float and clear, height and width, margin, min/max height and width, outline, overflow padding, position, text-decoration, verticle-align and z index.

* Inheritance prevents certain properties from being declared over and over again in a style sheet, allowing software developers to write less CSS. It also enhances faster loading of websites by users (more on the importance of avoiding repetition further below).

- To write a style that applies to the class description, we do so by typing:

.description {
Color: red;
}
(In this example, color serves as the attribute).

A further example would be if you wanted to write CSS that makes all h1 elements have a black background and white text, you would write:

h1 {
background-color: black;
color: white;
}

* IN THIS EXAMPLE *

1. The h1 is a Selector. It says to the browser, "I want you to apply the rules I'm about to tell you to every h1 element."
2. After the h1 there is a left curly brace ( { ) and at the bottom there is a matching right culy brace ( } ). Everything between these curly braces will be interpreted as a "rule" that should be applied to every h1 element.
3. The line of code that says: background-color: black; is a declaration. background-color is a property and black is the value of
that property.

In the example above, we set the background-color of every h1 to black. If we only want to add style to certain h1's, we can use Class Selectors. Example: The following Code -

<div class="stop"></div>
<div class="slow"></div>
<div class="go"></div>

<style>
div {
height: 50px;
width: 50px;
border-radius: 25px;
}
.stop {
background-color: red;
}
.slow {
background-color: yellow;
}
.go {
background-color: green;
}
</style>

This looks like a traffic light!!:

There are Three Ways to Include Style on a Webpage:

1. Write CSS in the <head> of your HTML. This method is good for very small projects. To do this:
* At the top of your HTML document, add a <head> element.
* Inside the <head> element, add a <style> element.
* Write your CSS in the <style> element and then put the HTML for the rest of your page below.

2. Link your HTML to a separate CSS file. This adds another step, but it lets you stay more organized when working on larger projects. To do this:
* Write all of your structural HTML in one file.
* In a separate file, put all of your CSS code.
* Add a <head> element to the top of your HTML.
* Add a <link> tag inside the <head> element. Since <link> is a "void tag", you don't need to add a closing </link>.
* Add the following attributes to your <link>: rel="stylesheet" type="text/css" href="main.css" (** if our CSS file was named
"main.css")

3. Write your style inline with your HTML. This is generally a very bad idea because it leads to lots of repeated code, but you may see code that uses this method so good to be familiar with it. With this method, you modify the style attributes of every individual HTML element.

CSS SUMMARY

- CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

* Additional Relevant Notes *

Code Comments - Lines of text computer ignores. Just there so that programmers can leave notes for other people.

*In HTML, code comments begin with <!-- and end with -->. Everything in between is ignored.
*In CSS, code comments begin with /* and end with */.

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.

Consistency: In the example provided, to make all lessons look the same, each lesson div was assigned the same class and then the style for that class was specified in one place (opposed to writing over and over again). This also makes it very easy to make changes to the overall style.

The Box Model

Content - Image or text appearing on the website.
Padding - Clears the area around the content. Affected by the background color of the box. The space inside of the box that "protects" the content directly.
Border - Inherited from the color property of the box.
Margin - Clears area around the border. Does not have a background color and is completely transparent. "The space between boxes".

* To make the calculation of the element size, include both the border and padding (so easier to make layouts). You can set the width of the overall box, and no matter how you tweak the padding and border sizes, the size of the box will always stay the same. You can do this in CSS by:

*{
- webkit-box-sizing:border-box;
- moz-box-sizing:border-box;
- ms-box-sizing:border-box;
box-sizing:border-box;
}

- Sizes of boxes are set in pixels (px) but it can also be done in percentage as well. You can also set a max width.

- Although not displayed in the image above, the box model also contains a Grid, which is displayed purely for visual reasons.

Box Sizing & Positioning Summary

1. HTML elements are boxes and each box has 4 components (margin, border, padding, content).
2. Because there are so many components to each box, it can often be hard to get the size of a box just right.
3. There are two techniques you can use to help with sizing issues:

*Set sizes in terms of percents rather than pixels.
*Set the box-sizing attribute to border-box for every element:

* {
box-sizing: border-box;
}

4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning

1. Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
2. Adding the rule <span =class"code">display:flex;</span> to the appropriate CSS will overide this behavior and let divs appear next to each other.
